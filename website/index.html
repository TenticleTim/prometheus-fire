BLOCKER · AI_013
Prompt Injection

AI generates code that concatenates user input directly into your prompts. Attackers can override your system prompt, extract secrets, or make your AI do anything.

// What Claude wrote ↓ const reply = await openai.chat.completions.create({ messages: [{ role: 'user', content: 'Summarize: ' + userInput }] }); ✗ BLOCKER — prompt_injection_user_input User input concatenated directly into prompt. Sanitize or use a structured message boundary.
thesmos/rules/ai.ts · AI_013
HIGH · ZOD_008
No Input Validation

AI skips Zod schemas on API routes because the code works locally. Unvalidated bodies reach your database. In production, this is a security vulnerability and a type-error time bomb.

// What Copilot wrote ↓ export async function POST(req: Request) { const body = await req.json(); await db.user.create({ data: body }); } ✗ HIGH — missing_request_body_schema No Zod schema on POST route. Unvalidated input reaches the database.
thesmos/rules/zod.ts · ZOD_008
HIGH · DB_003
N+1 Query in a Loop

AI writes clean-looking loops that fire a database query on every iteration. A list of 50 posts becomes 51 round-trips. Works fine in dev. Collapses under real traffic.

// What Cursor wrote ↓ for (const post of posts) { const author = await db.user.findUnique( { where: { id: post.authorId } } ); } ✗ HIGH — prisma_n_plus_one Prisma query inside a loop. 50 posts = 51 DB round-trips. Use include:.

New · Autopilot

The law that knows
where it's going.

Thesmos Autopilot executes your plan task by task, on its own git branch, under full governance — then reviews its own diff before you merge. You define what to build. Thesmos governs every decision along the way.

I · Autonomous Execution

Branch. Build. Journal.

Write your plan in Markdown. Type CONFIRMED to start. Thesmos creates an isolated git branch, executes each task in sequence, journals every AI decision, and stops safely the moment something goes wrong. Your main branch is never touched. Cancel anytime with a sentinel file or one click in VS Code.

thesmos autopilot start MASTER_PLAN.md

II · Plan Generator

Natural Language to Plan

Describe what you want to build in plain English. Thesmos reads your codebase, asks three clarifying questions, and generates a structured MASTER_PLAN.md — ready to validate and execute.

thesmos autopilot generate "add Stripe checkout"
III · Post-Session Review

The AI Reviews Its Own Diff

When the session ends, Thesmos reviews every change it made. It surfaces decisions it's uncertain about, summarizes the full diff, and gives you a structured merge recommendation — APPROVE, APPROVE WITH FIXES, or REQUEST CHANGES.

thesmos autopilot review [session-id]
IV · Convention Learning

Smarter Every Session

Every session teaches Thesmos more about your codebase. Preferred libraries, blocked packages, architectural patterns — stored in .thesmos/autopilot/conventions.json and applied to the next run automatically.

thesmos autopilot stats
V · Reconnaissance

Read Before You Write

Before touching a single file, Autopilot performs a read-only scan of the task's scope — discovering relevant files, mapping dependencies, and surfacing risks. Better context means fewer wrong turns.

thesmos autopilot start PLAN.md --recon

Setup

Governed in minutes.
Protected forever.

No account. No cloud. Thesmos runs locally, in CI, and in your editor — simultaneously. The feedback loop is both reactive and proactive. Autopilot extends it to autonomous.

I

Install

Run npm install --save-dev thesmos-governance. All 911 rules activate immediately — no config file needed.

II

Scan & Grade

Run thesmos scan to get your first health grade (A+ through F). Every finding is categorized by severity so you know exactly what to fix first.

III

Teach Your AI

Run thesmos adapters to generate governed briefs for Claude, Cursor, Copilot, and Gemini. Your AI now knows your rules before it writes a single line — governed from the first keystroke, not the last review.

IV

Gate Every PR

Drop one YAML file into .github/workflows/. Every pull request gets a full governance scan. BLOCKERs block the merge automatically — no human required to catch prompt injection or hardcoded secrets.

V

Launch Autopilot

Describe a feature in plain English: thesmos autopilot generate "add Stripe checkout". Answer three clarifying questions. Type CONFIRMED. Thesmos builds it autonomously — governed, journaled, and ready for your review.
