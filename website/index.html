AI tools make the same predictable mistakes across every project. Prometheus was built knowing exactly what they are — 911 rules derived from real production failures, not hypotheticals.

BLOCKER · AI_013
Prompt Injection

AI generates code that concatenates user input directly into your prompts. Attackers can override your system prompt, extract secrets, or make your AI do anything.

// What Claude wrote ↓ const reply = await openai.chat.completions.create({ messages: [{ role: 'user', content: 'Summarize: ' + userInput }] }); ✗ BLOCKER — prompt_injection_user_input User input concatenated directly into prompt. Sanitize or use a structured message boundary.
prometheus/rules/ai.ts · AI_013
HIGH · ZOD_008
No Input Validation

AI skips Zod schemas on API routes because the code works locally. Unvalidated bodies reach your database. In production, this is a security vulnerability and a type-error time bomb.

// What Copilot wrote ↓ export async function POST(req: Request) { const body = await req.json(); await db.user.create({ data: body }); } ✗ HIGH — missing_request_body_schema No Zod schema on POST route. Unvalidated input reaches the database.
prometheus/rules/zod.ts · ZOD_008
HIGH · DB_003
N+1 Query in a Loop

AI writes clean-looking loops that fire a database query on every iteration. A list of 50 posts becomes 51 round-trips. Works fine in dev. Collapses under real traffic.

// What Cursor wrote ↓ for (const post of posts) { const author = await db.user.findUnique( { where: { id: post.authorId } } ); } ✗ HIGH — prisma_n_plus_one Prisma query inside a loop. 50 posts = 51 DB round-trips. Use include:.
prometheus/rules/database.ts · DB_003

The Governing Mind
New · Autopilot

The fire that knows
where it's going.

Prometheus Autopilot executes your plan task by task, on its own git branch, under full governance — then reviews its own diff before you merge. You define what to build. Prometheus governs every decision along the way.

I · Autonomous Execution

Branch. Build. Journal.

Write your plan in Markdown. Type CONFIRMED to start. Prometheus creates an isolated git branch, executes each task in sequence, journals every AI decision, and stops safely the moment something goes wrong. Your main branch is never touched. Cancel anytime with a sentinel file or one click in VS Code.

prometheus autopilot start MASTER_PLAN.md

II · Plan Generator

Natural Language to Plan

Describe what you want to build in plain English. Prometheus reads your codebase, asks three clarifying questions, and generates a structured MASTER_PLAN.md — ready to validate and execute.

prometheus autopilot generate "add Stripe checkout"
III · Post-Session Review

The AI Reviews Its Own Diff

When the session ends, Prometheus reviews every change it made. It surfaces decisions it's uncertain about, summarizes the full diff, and gives you a structured merge recommendation — APPROVE, APPROVE WITH FIXES, or REQUEST CHANGES.

prometheus autopilot review [session-id]
IV · Convention Learning

Smarter Every Session

Every session teaches Prometheus more about your codebase. Preferred libraries, blocked packages, architectural patterns — stored in .prometheus/autopilot/conventions.json and applied to the next run automatically.

prometheus autopilot stats
V · Reconnaissance

Read Before You Write

Before touching a single file, Autopilot performs a read-only scan of the task's scope — discovering relevant files, mapping dependencies, and surfacing risks. Better context means fewer wrong turns.

prometheus autopilot start PLAN.md --recon

Run npm install --save-dev prometheus-governance. All 911 rules activate immediately — no config file needed.

Run prometheus scan to get your first health grade (A+ through F). Every finding is categorized by severity so you know exactly what to fix first.

III

IV

V

Launch Autopilot
