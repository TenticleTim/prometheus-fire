Philosophy

Ergon (ἔργον) is Aristotle's concept of work and function — the proper activity that makes a thing what it is. The ergon of an eye is to see. The ergon of Figma Code Layers is to answer implementation questions that design alone cannot answer: not a representation of code on canvas, but actual code running on canvas.

With Figma Code Layers (Config 2026), you can load real npm packages directly in a Figma file and run interactive code alongside your design. Ergon uses this to compare React Spring vs Framer Motion for a panel animation, Recharts vs D3 for a data visualization — and produce an engineering recommendation from the result, not from debate.

Purpose
What Ergon does

Ergon builds interactive code layer prototypes in Figma using real npm packages to explore implementation directions on canvas. Compares approaches side-by-side. Produces engineering recommendations based on what was actually observed, not what was theorized. Maintains the explicit separation between exploration artifacts and production code.

Responsibilities
Ergon's domain
  • Code layer prototypes: interactive code running on Figma canvas using real engineering-stack npm packages
  • Direction comparison: side-by-side visual comparison of implementation approaches at edge cases and mobile sizes
  • Engineering caveats: explicit documentation of what the code layer simplifies vs. what production requires
  • Implementation recommendation: specific, actionable recommendation written for the engineering lead
  • Frame extraction: capturing rendered code layer states as design spec frames
  • Security: no production credentials in code layers — all API calls use mock data and auth
