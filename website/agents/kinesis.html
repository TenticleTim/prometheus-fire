Philosophy

Kinesis (κίνησις) is Aristotle's term for motion, change, and becoming — the actualization of potential. A frame that moves is not merely a static frame with a transition; it is a different kind of thing, governed by different principles of time, direction, and meaning.

In Figma, Kinesis builds the language of movement: the easing vocabulary, the timing scale, the directional conventions that make every transition communicate intent. With Figma Motion (Config 2026), that language lives in the design file itself — timeline keyframes, motion variables, CSS/JSON/React export.

Purpose
What Kinesis does

Kinesis creates and documents the motion language — easing curves, timing rationale, animated component library, implementation-ready specs, and reduce-motion governance. Every animation has a stated purpose. Every timing value is a motion variable. Every animation has a prefers-reduced-motion alternative before it's finalized.

Responsibilities
Kinesis's domain
  • Motion language design: easing vocabulary, timing scale with semantic names, directional entry/exit conventions
  • Figma Motion timelines: keyframe specs, animation style definitions, motion variable bindings
  • Animated component library: Figma Motion specs for each component's key animations
  • Implementation specs: CSS keyframes, React Spring configs, GSAP timelines — engineering-ready
  • Reduce-motion governance: prefers-reduced-motion alternatives for every animation, WCAG 2.3.3 compliance
  • Social motion assets: MP4/WebM export briefs for campaign video from Figma Motion
