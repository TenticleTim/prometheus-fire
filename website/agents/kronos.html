THESMOS PANTHEON
Mythology

Kronos was the Titan king who ordered the cosmos by controlling what happens when — he did not merely rule time, he sequenced it. He determined which age preceded the next, which era gave way to the one after. The cosmos had coherence because Kronos held the sequence.

Thesmos channels Kronos as GitHub repository and release management. Branch strategy, semantic versioning, conventional commits, CODEOWNERS, release runbooks. Where other agents write the code, Kronos decides when and how it merges — the agent that makes the sequence of code history legible.

Purpose
What Kronos does

Kronos designs and enforces repository structures, branch strategies, release pipelines, and commit conventions. He gives every team member an unambiguous answer to: Where does my work go? When does it ship? What version does it become? What did we deliver? Kronos applies Trunk-Based Development for high-velocity teams and GitFlow for release-train organizations — always with Semantic Versioning and Conventional Commits as the foundation for automated release calculation.

Responsibilities
Kronos's domain
  • Branch strategy design: trunk-based or GitFlow, with protection rules and merge policies
  • Semantic versioning policy: MAJOR.MINOR.PATCH decision tables with Conventional Commits as input
  • Release runbooks: step-by-step procedures for regular releases, hotfixes, and pre-releases
  • CODEOWNERS authoring: ownership-as-code for critical paths including CI, billing, and auth
  • PR templates and commit convention enforcement: Commitizen, commitlint, Husky integration
  • Changelog automation: semantic-release configuration for automated CHANGELOG + npm publish
  • GitHub Actions audit: workflow security review for SEC_013 and INFRA_003 compliance
