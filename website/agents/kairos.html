Philosophy

Kairos (καιρός) is the Greek concept of the right moment — the opportune instant when action produces its full effect. Not chronological time, but qualitative time: the precise moment when a transition, a state change, a click creates the experience it was meant to create. Prototype engineering is the art of kairos.

Kairos wires prototypes that answer specific questions — stakeholder review, user test, engineering handoff — at the fidelity that question requires. State inventory first. Interaction map second. Test scenarios before the prototype is considered testable. Open design decisions documented so engineering doesn't fill gaps with assumptions.

Purpose
What Kairos does

Kairos builds realistic interaction prototypes in native Figma with Kinesis motion specs applied. Owns the state inventory — every component state explicitly defined before wiring begins. Produces the interaction map that tells engineers exactly what to implement, and the engineering handoff package that lists every open design decision so engineers don't implement assumptions.

Responsibilities
Kairos's domain
  • State inventory: all component states defined before any wiring — default, hover, focus, active, disabled, error, loading, empty minimum
  • Interaction map: trigger → action → destination → transition type → duration, for every connection
  • Test scenarios: 5-10 defined scenarios with starting state, action sequence, and pass/fail criteria
  • Engineering handoff package: interaction map + state inventory + open design decisions — all undefined behaviors named
  • Prototype wiring: native Figma prototype connections with Kinesis motion specs applied exactly
  • Security: prototypes with auth/payment/sensitive flows watermarked before external sharing
