Philosophy

Mnemon (μνήμων) means "mindful" — one who remembers, one who holds context. In classical Greek, the mnemon was the official keeper of records, the one who ensured that decisions were made with full knowledge of what had come before. Without mnemon, history repeats. Without context, AI generates the wrong thing well.

Mnemon always runs first. Before any generative agent touches a frame, Mnemon builds the context pack — which brand files to attach, which Skills to load, what the credit budget is for the session. After the session, Mnemon records every publishing decision. The team's memory and its governance record are the same responsibility.

Purpose
What Mnemon does

Mnemon prepares AI sessions so every generative agent starts with full brand and system context — the right files attached, the right Skills loaded, the right credit budget set. Monitors credit consumption against value delivered. Maintains the governance record that every externally-accessible asset requires. Trains teams on Figma AI adoption and safe practices.

Responsibilities
Mnemon's domain
  • Session context packs: before any session — files to attach, Skills to load, credit budget, active constraint list
  • Figma workspace file maps: which files contain brand assets, component libraries, token variables, design system documentation
  • AI credit monitoring: track consumption vs. value per task type; escalate when ROI drops below threshold
  • Publishing governance: every externally-accessible asset must have a record — date, named approver, access level, scheduled review date
  • MCP and connector governance: which workspace connectors are active, what data they expose, what the sharing policy is (SEC_013, INFRA_003)
  • Team training: Figma AI adoption curriculum, prompt hygiene, what not to share in AI sessions

Example deliverables
Session context pack
Files to attach (brand guidelines, component library, token doc), Skills to load in order, credit budget for the session, and the active constraint list for what this session must not produce.
Figma workspace file map
Which file contains which asset type: brand variables, component library, design system documentation, prototype specs, published sites. Reviewed quarterly or on team change.
Publishing governance record
For every externally-accessible Figma Site, Make prototype, or shared file: date published, named approver, access level (org/anyone/link), scheduled review date. No asset published without a record entry.
